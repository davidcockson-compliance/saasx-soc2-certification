# Contributing Guidelines

## Workflow
1. Pick an issue from the project board
2. Assign yourself
3. Create a branch: `git checkout -b [type]/[description]`
4. Complete the work
5. Commit changes: `git commit -m "Description"`
6. Push branch: `git push origin [branch-name]`
7. Create pull request
8. Get review and approval
9. Merge to main

## Branch Naming
- `policy/[name]` - Policy documents
- `control/[name]` - Control implementation
- `evidence/[name]` - Evidence collection
- `docs/[name]` - Documentation updates

## File Naming
Use lowercase with hyphens:
- `information-security-policy.pdf`
- `access-logs-2025-10.csv`

## Questions?
Contact the project manager or post in issues.
